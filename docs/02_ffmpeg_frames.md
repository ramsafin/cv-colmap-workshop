# 02. Извлечение кадров из видео с помощью FFmpeg

На этом этапе из готового **видеофайла** делают **набор отдельных снимков (изображений)**, с которыми дальше работает COLMAP.

**Где выполнять команды:** внутри контейнера воркшопа, в каталоге `/workspace` (как после шага из [инструкции по подготовке окружения](01_setup.md)).

## Как пользоваться этим файлом

1. Прочитайте раздел **«Почему не берём все кадры из видео»** — чтобы понимать смысл редкой выборки.
2. Запустите **скрипт** `extract_frames.sh` (рекомендуется) или при необходимости скопируйте **ручную команду FFmpeg**.
3. Выполните **проверки** из раздела про результат.
4. Перейдите к следующему документу про признаки и сопоставление в COLMAP.

## Цель этапа

Нужно:

1. взять **входное видео** `sample.mp4` (или другое, если вы передадите свой путь в скрипт);
2. **извлечь** из него ограниченный набор **кадров** (стоп‑кадров);
3. **уменьшить разрешение** каждого снимка до **960×540** пикселей (меньше объём данных и быстрее шаги в COLMAP);
4. получить порядка **100–110** файлов с именами вида:
   - `frame_001.jpg`
   - `frame_002.jpg`
   - `frame_003.jpg`

Дальше эти снимки пойдут в COLMAP на этапы:

- **извлечение признаков** (характерных точек на изображениях);
- **поиск соответствий** (сопоставление точек между парами снимков);
- **разрежённая 3D-реконструкция** (англ. *sparse reconstruction* — восстановление грубого **облака точек** и положений камер).

Примеры видео можно взять с сайта набора данных [Tanks and Temples](https://www.tanksandtemples.org/download/) (название на английском; для воркшопа обычно используют один общий ролик `sample.mp4`).

---

## Почему не берём все кадры из видео

Видео — это очень плотная последовательность кадров: соседние кадры часто **почти одинаковы**.  
Если отдать в COLMAP **все** кадры подряд:

- сильно вырастет **время** обработки;
- появится много **лишних** почти дублирующих снимков;
- этап **сопоставления** пар изображений (в учебном сценарии с полным перебором пар) станет **медленнее**;
- польза от сотен «одинаковых» ракурсов для модели будет невелика.

Поэтому мы:

- **реже выбираем** кадры из ролика (регулируется параметром **`fps`** в фильтре FFmpeg — не путать с «кадрами в секунду исходного файла», это именно **частота извлечения** новых снимков);
- получаем **компактный** набор снимков, удобный для занятия;
- стараемся сохранить **достаточное перекрытие сцены** (англ. *overlap*) — общую видимую область между **соседними** извлечёнными кадрами, чтобы COLMAP мог связать их в одну модель.

---

## Целевой результат

После шага в каталоге для снимков:

```text
/workspace/data/images
```

должны появиться файлы в формате **JPEG** (сжатые фотографии):

```text
frame_001.jpg
frame_002.jpg
frame_003.jpg
...
```

Ожидаемое **число** файлов для эталонного `sample.mp4` и настроек по умолчанию:

* ориентировочно около **110** штук;
* немного больше или меньше — нормально при **другой длительности** ролика или **другой частоте извлечения** в фильтре `fps`.

## Ручная команда FFmpeg (справочно)

Ниже — одна строка разбита на несколько для читаемости. Её **не обязательно** набирать от руки: удобнее вызвать скрипт из следующего раздела — он делает то же самое и ещё **очищает** старые `frame_*.jpg` перед записью.

```bash
ffmpeg -y \
  -i /workspace/data/video/sample.mp4 \
  -vf "fps=3/4,scale=960:540" \
  -q:v 2 \
  -start_number 1 \
  /workspace/data/images/frame_%03d.jpg
```

### Что означают ключи

* `-y` — **перезаписать** выходные файлы без вопроса «файл уже существует»;
* `-i путь` — **входной** файл (здесь — видео);
* `-vf "…"` — **цепочка видеофильтров** (англ. *video filter*):

  * `fps=3/4` — в среднем **0,75 извлечённого снимка на каждую секунду** исходного видео (то есть 3 снимка за 4 секунды записи); для типичного `sample.mp4` это даёт порядка сотни файлов и сохраняет **перекрытие** между соседними снимками;
  * `scale=960:540` — подогнать размер к **960×540 пикселей** (часто говорят «разрешение qHD», четверть от Full HD 1920×1080);
* `-q:v 2` — **качество сжатия JPEG** (меньше число — выше качество и больше размер файла; `2` — хороший компромисс для воркшопа);
* `-start_number 1` — нумерация имён **с единицы**, а не с нуля;
* `frame_%03d.jpg` — шаблон имени: `%03d` значит «номер из **трёх** цифр с ведущими нулями» → `001`, `002`, ….

## Рекомендуемый способ: скрипт

В проекте есть готовый сценарий:

```text
scripts/extract_frames.sh
```

Запуск **из каталога** `/workspace` внутри контейнера:

```bash
bash scripts/extract_frames.sh
```

Справка по опциям: `bash scripts/extract_frames.sh --help` (или `-h`).

Перед записью скрипт **по умолчанию удаляет** старые файлы `frame_*.jpg` в выходной папке, чтобы не смешивать результаты разных запусков. Чтобы **не** удалять их, передайте **`--no-cleanup`** (см. `--help`).

По умолчанию (см. исходный текст [`scripts/extract_frames.sh`](../scripts/extract_frames.sh)) задано:

| Параметр | Значение по умолчанию |
|-----------|------------------------|
| входное видео | `/workspace/data/video/sample.mp4` |
| папка для снимков | `/workspace/data/images` |
| частота извлечения `fps` | `3/4` (дробь в терминах фильтра FFmpeg) |
| ширина × высота | `960` × `540` |
| качество JPEG (`-q:v`) | `2` |

---

## Запуск с параметрами

Те же шесть значений можно задать **флагами** (`-i`, `-o`, `--fps`, …) — см. `bash scripts/extract_frames.sh --help`. Ниже — **позиционный** вариант (как в прежних версиях воркшопа): шесть аргументов подряд или после `--`.

Пример: то же видео и папка, но **чаще** брать кадры (`fps=1` — в среднем один снимок на секунду исходного видео):

```bash
bash scripts/extract_frames.sh -- \
  /workspace/data/video/sample.mp4 \
  /workspace/data/images \
  1 \
  960 \
  540 \
  2
```

Эквивалентно без `--` (если не используете короткие/длинные опции):

```bash
bash scripts/extract_frames.sh \
  /workspace/data/video/sample.mp4 \
  /workspace/data/images \
  1 \
  960 \
  540 \
  2
```

Здесь третий аргумент `1` задаёт фильтр `fps=1`; для настроек «как в скрипте из коробки» достаточно вызвать `bash scripts/extract_frames.sh` **без** аргументов.

**Порядок аргументов** (все шесть можно не указывать — тогда берутся значения по умолчанию из таблицы выше):

1. путь к **входному видео**;
2. **выходная папка** для `frame_*.jpg`;
3. значение для фильтра **`fps=...`** (дробь передаётся **без кавычек**, например `3/4` или целое `1`);
4. **ширина** в пикселях;
5. **высота** в пикселях;
6. **качество JPEG** для `-q:v` (чем меньше число, тем выше качество; `0` — почти без потерь по таблице квантования, файл будет тяжелее).

## Проверка результата

### Сколько получилось снимков

```bash
find /workspace/data/images -maxdepth 1 -type f -name 'frame_*.jpg' | wc -l
```

Число в последней строке — количество файлов `frame_*.jpg` **только** в этой папке (без подпапок).

### Имена в начале списка

```bash
ls /workspace/data/images | head
```

### Имена в конце списка

```bash
ls /workspace/data/images | tail
```

### Есть ли первый кадр

```bash
ls /workspace/data/images/frame_001.jpg
```

Если файла нет, команда сообщит об ошибке (в т.ч. текст вроде *No such file* — «нет такого файла»): значит, нумерация не с единицы или извлечение не сработало.

---

## Что считаем успехом

* каталог `/workspace/data/images` существует;
* в нём есть хотя бы один файл `frame_*.jpg`;
* первый файл — **`frame_001.jpg`** (как задал `-start_number 1` и шаблон имён);
* число снимков **не ноль** и выглядит разумно для вашего видео и выбранного `fps`.

## Возможные проблемы

### Проблема 1. Снимки не появились

Проверьте:

* существует ли **входной** файл видео по указанному пути;
* нет ли опечатки в пути (учитывайте регистр букв в Linux);
* не вывел ли **FFmpeg** в консоль сообщение об ошибке (кодек, повреждённый файл, нет места на диске).

### Проблема 2. Снимков слишком мало

Возможные причины:

* **короткий** исходный ролик при той же частоте извлечения;
* слишком **низкая** частота в фильтре `fps` (слишком редко берёте кадры);
* ошибка в **аргументах** скрипта (перепутан порядок ширины и высоты и т.п.).

Попробуйте **увеличить** частоту извлечения (например, передать третьим аргументом `1` вместо `3/4`).

### Проблема 3. Снимков слишком много

Возможные причины:

* слишком **высокая** частота в `fps` для длинного видео;
* очень **длинный** ролик при `fps=1` или выше.

Попробуйте **уменьшить** частоту извлечения (например, вернуть `3/4` или указать дробь поменьше).

### Проблема 4. В папке остались старые `frame_*.jpg`

По умолчанию скрипт `extract_frames.sh` перед работой **удаляет** старые `frame_*.jpg` в выбранной выходной папке. Если нужно **сохранить** старые файлы и дописать новые рядом, вызовите скрипт с **`--no-cleanup`** (см. **`bash scripts/extract_frames.sh --help`**).

Если вы запускали **только** длинную команду `ffmpeg` вручную и меняли шаблон имён, старые файлы могли **не затронуться** при запуске скрипта по умолчанию — тогда удалите лишнее вручную или снова запустите скрипт на ту же папку **без** **`--no-cleanup`**.

## Что на этом шаге сознательно не делаем

Чтобы не распылять внимание, **пока не** выполняем:

* автоматическую **оценку качества** снимков;
* **отбор** «слишком похожих» кадров и их удаление;
* **запуск COLMAP** (это следующие файлы инструкции).

# 01. Setup: Docker, image import, container run

Этот документ описывает подготовку окружения для воркшопа по COLMAP.

## Цель этапа

На этом шаге нужно:

1. получить Docker image воркшопа;
2. импортировать готовый образ **или** собрать его вручную;
3. запустить контейнер;
4. проверить, что внутри доступны:
   - `colmap`
   - `ffmpeg`
   - `python`
5. убедиться, что рабочие директории и входное видео видны из контейнера.

---

## Что должно быть установлено на хосте

На компьютере студента должны быть установлены:

- Docker
- Docker Compose

### Поддерживаемые платформы

- Linux (Ubuntu)
- macOS (Apple Silicon / ARM)
- Windows + Docker Desktop + WSL2 + Ubuntu

---

## Структура проекта

Предполагается, что репозиторий уже распакован локально и содержит такую структуру:

```text
cv-colmap-workshop/
├─ README.md
├─ compose.yaml
├─ docker/
├─ docs/
├─ scripts/
└─ data/
   ├─ video/
   │  └─ sample.mp4
   ├─ images/
   └─ workspace/
```

Перед запуском убедитесь, что файл входного видео существует:

```text
data/video/sample.mp4
```

---

## Вариант A. Импорт готового Docker image

Это основной путь для воркшопа.

### Шаг 1. Скопировать архив с образом

С флешки нужно скопировать архив Docker image, например:

```text
colmap-workshop-image.tar
```

в любую удобную директорию.

### Шаг 2. Импортировать образ

Из терминала выполнить:

```bash
docker image load -i colmap-workshop-image.tar
```

### Шаг 3. Проверить, что образ появился

```bash
docker images
```

В списке должен появиться образ воркшопа.

Пример ожидаемого результата:

```text
REPOSITORY          TAG       IMAGE ID       CREATED         SIZE
colmap-workshop     latest    <hash>         <date>          <size>
```

---

## Вариант B. Сборка образа вручную

Используйте этот путь, если готового архива нет или требуется локальная пересборка.

Из корня репозитория выполните:

```bash
docker compose build
```

Сборка может занять некоторое время (обычно требуется подключение к интернету).

**Внимание:** не рекомендуется использовать мобильный интернет, размер файлов-образов может достигать несколько гигабайт.

После успешной сборки проверьте список образов:

```bash
docker images
```

---

## Запуск контейнера

Из корня репозитория выполните:

```bash
docker compose run --rm workshop bash
```

После запуска вы должны оказаться внутри контейнера в рабочей директории проекта.

Пример приглашения shell:

```text
root@<container-id>:/workspace#
```

---

## Что должно быть примонтировано внутрь контейнера

Внутри контейнера должны быть доступны:

* `/workspace/README.md`
* `/workspace/docs/`
* `/workspace/scripts/`
* `/workspace/data/video/sample.mp4`
* `/workspace/data/images/`
* `/workspace/data/workspace/`

Проверка:

```bash
pwd
ls
ls /workspace
ls /workspace/data/video
```

Ожидается, что команда покажет файл:

```text
sample.mp4
```

## Ручная проверка инструментов (вместо `smoke_test.sh`)

### Проверка COLMAP

```bash
colmap help
```

Ожидаемый результат:

* команда выполняется без ошибки;
* отображается список доступных команд COLMAP.

### Проверка FFmpeg

```bash
ffmpeg -version
```

Ожидаемый результат:

* отображается версия FFmpeg.

### Проверка Python

```bash
python --version
```

или

```bash
python3 --version
```

Ожидаемый результат:

* отображается версия Python.

### Проверка видео

```bash
ls /workspace/data/video/sample.mp4
```

Ожидаемый результат:

* путь к файлу отображается без ошибки.

---

## Проверка прав записи

Внутри контейнера выполните:

```bash
touch /workspace/data/workspace/.write_test
ls /workspace/data/workspace
```

Затем удалите тестовый файл:

```bash
rm /workspace/data/workspace/.write_test
```

Если команды проходят без ошибки, рабочая директория доступна для записи.

---

## Минимальный чек-лист для проверки

Считаем этап setup успешным, если:

* Docker image импортирован или собран;
* контейнер запускается;
* `colmap help` работает;
* `ffmpeg -version` работает;
* `python --version` работает;
* файл `sample.mp4` доступен внутри контейнера;
* директории `data/images` и `data/workspace` доступны для записи.

## Рекомендуемый порядок действий на воркшопе

1. открыть терминал;
2. перейти в корень репозитория;
3. импортировать готовый образ или собрать его;
4. запустить контейнер;

## Заметки

### Linux (Ubuntu)

Обычно достаточно:

```bash
docker compose build
docker compose run --rm workshop bash
```

или импорта готового image и запуска контейнера.

### macOS (Apple Silicon / ARM)

Рекомендуется использовать Docker Desktop.

Запуск аналогичен:

```bash
docker compose run --rm workshop bash
```

### Windows (Docker Desktop + WSL2)

Рекомендуемый сценарий:

1. установить Docker Desktop;
2. включить WSL2;
3. использовать Ubuntu в WSL;
4. запускать команды из терминала WSL.

Из терминала Ubuntu:

```bash
docker compose run --rm workshop bash
```

## Типичные проблемы

### Проблема 1. `docker compose` не найден

Проверьте, установлен ли Docker Compose.

Попробуйте:

```bash
docker compose version
```

### Проблема 2. Контейнер не запускается

Проверьте:

* Docker daemon запущен;
* образ существует;
* команда запуска выполняется из корня репозитория.

### Проблема 3. В контейнере не видно `sample.mp4`

Проверьте:

* существует ли файл `data/video/sample.mp4` на хосте;
* корректно ли примонтирован проект в контейнер;
* запускается ли `docker compose` из корня репозитория.

### Проблема 4. Нет прав на запись в `data/workspace`

Проверьте:

* права доступа к директории на хосте;
* не открыт ли проект в режиме только чтения;
* корректно ли работает bind mount.

### Проблема 5. `colmap` или `ffmpeg` не найдены

Это означает, что:

* образ собран некорректно;
* был импортирован не тот image;
* внутри контейнера используется не тот PATH или не тот базовый образ.

В этом случае нужно пересобрать образ или использовать корректный архив образа.

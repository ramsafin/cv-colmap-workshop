# 04. COLMAP: разрежённая 3D-реконструкция командой `mapper`

Здесь по снимкам, признакам и сопоставлениям из **`database.db`** восстанавливается **разрежённая** (не плотная) **трёхмерная модель сцены**: положения камер и **облако точек** (*sparse point cloud*).

**Где выполнять:** контейнер, каталог `/workspace`. Нужны готовые [признаки и сопоставление](03_colmap_features_matching.md).

## Как пользоваться этим файлом

1. Убедитесь, что есть **`/workspace/data/images`** и **`/workspace/data/workspace/database.db`** после шага 03.
2. Запустите **`mapper`** через скрипт или вручную (команды ниже). Режим **`mapping`** и переменные: **`bash scripts/run_colmap_sparse.sh --help`**.
3. Проверьте папку **`sparse/0`** и журнал **`mapper.log`**.
4. При необходимости разберите вывод **`model_analyzer`** (метрики качества).
5. Дальше — [05_export_and_inspect.md](05_export_and_inspect.md).

На этом шаге строится **разрежённая 3D-реконструкция** (англ. *sparse reconstruction*, метод **SfM** — *structure from motion*, «структура из движения» камеры) на основе:

- снимков;
- извлечённых **признаков**;
- найденных **соответствий** между снимками.

Используется команда **`colmap mapper`**.

## Цель этапа

В результате вы получаете:

- каталог **разрежённой модели** (часто **`sparse/0`**);
- **зарегистрированные** в модели снимки (англ. *registered images* — кадры, которые COLMAP «привязал» к сцене);
- **положения и ориентации камер** (часто говорят **позы**, англ. *poses*);
- **разрежённое облако точек** — набор 3D-точек с цветами.

## Входные данные

Должны существовать:

```text
/workspace/data/images
/workspace/data/workspace/database.db
```

## Ручная команда `mapper`

```bash
mkdir -p /workspace/data/workspace/sparse

colmap mapper \
  --database_path /workspace/data/workspace/database.db \
  --image_path /workspace/data/images \
  --output_path /workspace/data/workspace/sparse
```

---

## Запуск через скрипт

Только шаг **`mapper`**:

```bash
bash scripts/run_colmap_sparse.sh mapping
```

Полный цикл «с нуля» (признаки → сопоставление → **`mapper`**):

```bash
bash scripts/run_colmap_sparse.sh all
```

Справка по скрипту (режимы, переменные окружения, **`--no-version-warn`**): **`bash scripts/run_colmap_sparse.sh --help`**.

---

## Что делает `mapper`

**`mapper`** собирает **геометрию сцены**: какие 3D-точки видны с каких снимков и где находились «камеры» для каждого снимка. Результат — один или несколько подкаталогов в:

```text
/workspace/data/workspace/sparse
```

Обычно появляется хотя бы одна модель, чаще всего с именем **`0`**:

```text
/workspace/data/workspace/sparse/0
```

---

## Что должно появиться после успешного запуска

```text
/workspace/data/workspace/sparse/0
/workspace/data/workspace/logs/mapper.log
```

## Проверка результата

### Подкаталоги моделей

```bash
find /workspace/data/workspace/sparse -mindepth 1 -maxdepth 1 -type d
```

### Содержимое первой модели (если это `0`)

```bash
ls -la /workspace/data/workspace/sparse/0
```

### Журнал

```bash
tail -n 30 /workspace/data/workspace/logs/mapper.log
```

### Краткая статистика модели

```bash
colmap model_analyzer --path /workspace/data/workspace/sparse/0
```

Пояснение полей (вывод на английском, смысл по-русски):

| В выводе | Что означает |
|----------|----------------|
| **Registered images** | сколько снимков **привязано** к сцене (желательно близко к числу всех снимков) |
| **Points** | сколько **3D-точек** в модели |
| **Mean track length** | в среднем сколько **снимков** «видит» каждая 3D-точка: \< 3 — слабо; 3–6 — нормально; \> 6 — хорошо |
| **Mean reprojection error** | средняя **ошибка перепроецирования** в пикселях: \< 0,5 — отлично; 0,5–1 — хорошо; 1–2 — терпимо; \> 2 — плохо, возможны проблемы калибровки или данных |
| **Observations** | сколько всего **наблюдений** точек на снимках |
| **Mean observations per image** | в среднем сколько 3D-точек «видно» на одном снимке |

## Критерии успеха

* **`mapper`** завершился без ошибки (см. лог);
* есть каталог **`sparse/0`** (или другой номер — тогда используйте его в экспорте);
* внутри — файлы бинарной модели COLMAP;
* можно переходить к **экспорту**: [05_export_and_inspect.md](05_export_and_inspect.md).

## Возможные проблемы

### 1. Нет папки `sparse/0`

* не выполнены или с ошибкой **извлечение признаков** и **сопоставление**;
* снимки **плохо подходят** для реконструкции (мало текстуры, мало **перекрытия** сцены между кадрами);
* мало надёжных **соответствий** (англ. *correspondences*).

### 2. Модель «слабая» (мало точек или мало зарегистрированных снимков)

* мало **перекрытия** между соседними кадрами;
* кадры почти одинаковые или, наоборот, слишком далеко друг от друга;
* мало **текстуры** на объектах;
* много **движущихся объектов** в кадре;
* неудачное исходное видео для **SfM**.

### 3. Несколько папок `sparse/0`, `sparse/1`, …

Это не всегда ошибка: COLMAP мог выделить **несколько несвязных фрагментов** сцены. Для воркшопа часто берут **`sparse/0`**. Если фрагментов много — улучшайте данные или сопоставление (см. [06_troubleshooting.md](06_troubleshooting.md)).

## Что на этом шаге сознательно не делаем

* **плотную** реконструкцию (*dense reconstruction*);
* **поверхностную сетку** (*mesh*);
* **графический интерфейс** COLMAP;
* подробный разбор **каждого** внутреннего файла модели.

Это темы для самостоятельного изучения или [обзора дальнейших шагов](07_next_steps.md).

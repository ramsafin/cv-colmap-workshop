# 06. Устранение типичных неполадок

Этот файл помогает **быстро найти этап**, на котором сломался воркшоп, и **что проверить** в первую очередь.

## Как пользоваться этим документом

Если **цепочка шагов** не срабатывает, не меняйте сразу всё подряд. Сначала определите **номер этапа** сбоя:

1. подготовка окружения / Docker;
2. извлечение кадров (FFmpeg);
3. извлечение признаков (**`feature_extractor`**);
4. сопоставление (**`matching`**);
5. разрежённая реконструкция (**`mapper`**);
6. экспорт и просмотр.

Дальше откройте соответствующий раздел ниже или пройдите **порядок из раздела 10** с конца файла.

---

## 1. Docker и окружение

### Симптом

Контейнер не запускается или внутри нет нужных программ.

### Что проверить

```bash
docker compose config
docker compose build
docker compose run --rm workshop bash
docker compose run --rm workshop bash scripts/smoke_test.sh
```

### Норма

* `docker compose config` без ошибок;
* контейнер стартует;
* **`smoke_test.sh`** завершается успешно.

### Если не работает

* запущена ли **служба Docker** (на Windows — Docker Desktop в состоянии «Running»);
* команды выполняются из **корня проекта** (рядом с **`compose.yaml`**);
* на диске есть **`data/video/sample.mp4`**;
* файл **`compose.yaml`** не меняли случайно.

## 2. Извлечение кадров

### Симптом

В **`data/images`** нет файлов **`frame_*.jpg`**, или их слишком мало / слишком много.

### Что проверить

```bash
bash scripts/extract_frames.sh
find /workspace/data/images -maxdepth 1 -type f -name 'frame_*.jpg' | wc -l
ls /workspace/data/images | head
ls /workspace/data/images | tail
```

### Норма

Для эталонного видео при **`fps=3/4`** и размере **960×540** ожидается около **110** снимков.

### Если снимков нет

* есть ли входное видео и верный ли путь;
* сообщения об ошибке **FFmpeg** в консоли;
* не перенаправили ли вы вывод в другую папку.

### Если снимков слишком много / мало

Подстройте **третий аргумент** скрипта (частота в фильтре **`fps`**): уменьшите — меньше снимков, увеличьте — больше.

### Если снимки есть, а реконструкция потом плохая

Часто виновато **содержание видео**, а не FFmpeg: мало текстуры, сильное **размытие в движении** (*motion blur*), мало **перекрытия** сцены, много движущихся объектов, камера почти не двигается.

## 3. Не создаётся `database.db` после признаков

### Симптом

После **`features`** нет **`database.db`** или файл пустой.

### Что проверить

```bash
bash scripts/run_colmap_sparse.sh features
ls -lh /workspace/data/workspace/database.db
tail -n 30 /workspace/data/workspace/logs/feature_extractor.log
```

### Возможные причины

* пустая **`data/images`**;
* снимки лежат не в **`/workspace/data/images`**;
* неверный **`--database_path`** в ручной команде.

### Что делать

1. Проверить снимки ([шаг 02](02_ffmpeg_frames.md)).
2. Читать **`feature_extractor.log`**.
3. Повторить **`features`**.
4. Убедиться в путях **`--image_path`** и **`--database_path`**.

---

## 4. Сопоставление слишком долгое

### Симптом

**`exhaustive_matcher`** идёт очень долго.

### Почему

Он сравнивает **каждую пару** снимков. Для **видео** COLMAP часто рекомендует **`sequential_matcher`**: учитывается порядок кадров, в первую очередь соседи. Также помогают **меньше признаков** и меньший размер изображения для SIFT.

### Что сделать в первую очередь

Для длинного видео попробуйте вручную (после **`features`**, до **`mapper`**):

```bash
colmap sequential_matcher \
  --database_path /workspace/data/workspace/database.db \
  --SiftMatching.use_gpu 0 \
  --SequentialMatching.overlap 10
```

### Если нужен именно полный перебор пар

Уменьшите **`--SiftExtraction.max_image_size`** и **`--SiftExtraction.max_num_features`**, либо снимите лишние кадры на шаге 02.

### Практическая памятка

* для **длинного видео** в реальных задачах чаще — **`sequential_matcher`**;
* в **этом воркшопе** в скрипте зафиксирован **`exhaustive_matcher`** для наглядности.

## 5. Сопоставление падает с ошибкой

### Симптом

**`matching`** не доходит до конца или завершается с ошибкой.

### Что проверить

```bash
bash scripts/run_colmap_sparse.sh matching
tail -n 30 /workspace/data/workspace/logs/exhaustive_matcher.log
```

(Если вы перешли на **`sequential_matcher`**, смотрите вывод этой команды в консоли.)

### Возможные причины

* не выполнен **`feature_extractor`**;
* нет или повреждена **`database.db`**;
* база пересоздана между шагами, а снимки менялись.

### Что делать

1. Удалить старую **`database.db`** (при необходимости сделайте копию).
2. Заново **`features`**, затем **`matching`**.
3. Между шагами **не менять** набор файлов в **`images`**.

## 6. Нет каталога `sparse/0` после `mapper`

### Симптом

После **`mapper`** не появилась папка **`sparse/0`**.

### Что проверить

```bash
bash scripts/run_colmap_sparse.sh mapping
tail -n 50 /workspace/data/workspace/logs/mapper.log
find /workspace/data/workspace/sparse -mindepth 1 -maxdepth 1 -type d
```

### Возможные причины

* сопоставление не завершилось или дало мало **соответствий**;
* видео плохо подходит для **SfM**;
* мало **перекрытия** или текстуры.

### Что делать

1. Убедиться, что **`matching`** успешен.
2. Попробовать **`sequential_matcher`** (раздел 4).
3. Увеличить **перекрытие** (чаще брать кадры на шаге 02).
4. Взять другое видео.

## 7. Несколько моделей: `sparse/0`, `sparse/1`, …

### Симптом

Появилось несколько подкаталогов с номерами.

### Что это значит

Не всегда ошибка: COLMAP мог выделить **несколько слабо связанных частей** сцены.

### Что делать

Для отчёта по воркшопу обычно берут **`sparse/0`**. Если частей много — улучшайте перекрытие и качество сопоставления.

---

## 8. Не экспортируются TXT или PLY

### Симптом

**`model_converter`** не создал файлы.

### Что проверить

```bash
bash scripts/export_model.sh
ls -la /workspace/data/workspace/text
ls -lh /workspace/data/workspace/ply/model.ply
```

### Возможные причины

* неверный путь к каталогу модели;
* **`mapper`** фактически не построил модель;
* выбран не тот подкаталог **`sparse/N`**.

Описание форматов — в [официальной документации COLMAP](https://colmap.github.io/).

### Что делать

1. Проверить **`sparse/0`** (или ваш номер).
2. Убедиться, что каталог модели не пустой.
3. Повторить **`export_model.sh`**.

## Краткий итог

Если **реконструкция** не получилась, чаще всего:

* плохие **входные кадры**;
* неподходящий или слишком медленный режим **сопоставления**;
* слабая **связность** набора снимков (мало общего между кадрами).

Практичный набор для видео: разрешение **960×540**, порядка **110** кадров, для длинных роликов — **`sequential_matcher`**.
